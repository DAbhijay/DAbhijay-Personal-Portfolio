PORTFOLIO WEBSITE — OVERVIEW (MULTI-PAGE)
========================================

GOAL
----
Build a clean, professional multi-page portfolio website that showcases my
frontend skills (HTML, CSS, JavaScript) and highlights 2–3 strong projects.
The site should be fast, accessible, responsive, and easy to extend.


TARGET AUDIENCE
---------------
- Recruiters / hiring managers
- Other developers
- Potential freelance clients (secondary)


SITE PAGES
----------
1. Home
   - Short intro / hero
   - Clear value statement
   - Links to Projects and About
   - Minimal and focused

2. About
   - Professional background
   - Skills summary
   - What I’m currently learning / focusing on
   - Optional photo

3. Projects
   - List of featured projects
   - Each project card includes:
     - Title
     - Short description
     - Tech stack
     - Screenshot
     - Links (GitHub / Live demo)

4. Project Detail Page (optional but recommended)
   - Deep dive per project
   - Problem → solution → outcome
   - Challenges & learnings
   - Screenshots

5. Contact
   - Email
   - GitHub
   - LinkedIn
   - Optional contact form


NAVIGATION STRUCTURE
--------------------
- Persistent header navigation
- Same nav across all pages
- Clear active state for current page
- Footer with repeated contact links


TECH STACK
----------
- HTML (semantic, accessible)
- CSS (custom properties, Flexbox/Grid)
- JavaScript (vanilla, modular)
- JSON / JS objects for project data
- No framework unless it clearly reduces complexity


PROJECT STRUCTURE
-----------------
/root
  ├── index.html          (Home)
  ├── about.html
  ├── projects.html
  ├── project.html        (single project template)
  ├── contact.html
  ├── css/
  │     └── styles.css
  ├── js/
  │     ├── data.js       (projects data)
  │     ├── nav.js        (shared navigation logic)
  │     └── main.js
  ├── assets/
  │     └── images/
  └── overview.txt


DESIGN PRINCIPLES
-----------------
- Content-first layout
- Consistent structure across pages
- Minimal color palette
- Strong typography hierarchy
- Reusable layout patterns
- Mobile-first approach


WHAT TO WORK ON NEXT (ORDER)
----------------------------
1. Define page responsibilities (what each page contains)
2. Write content for all pages
3. Create shared header + footer
4. Build Home page markup
5. Build Projects listing page
6. Build About page
7. Add project detail page logic
8. Apply global CSS system
9. Make site responsive
10. Add JS enhancements
11. Test, polish, deploy


NON-GOALS
---------
- No SPA routing
- No unnecessary animations
- No overengineering
- No redesigning after structure is locked


FUTURE IMPROVEMENTS
-------------------
- Dark mode
- Blog / writing page
- Static site generator
